                  NOTE: 3.7 is an alpha version, which fixes many issues, but may also introduce new ones. Please do not hesitate to add them to the issue tracker (https://github.com/supercollider/supercollider/issues) or on the supercollider mailing list.

                  This document focusses on features, there are many bugfixes and interesting improvements, which can be found by exploring the recent changes file and snooping around the system itself.

                  Editor

                  • Menu entries for Recording, scope and server inspection
                  • Modify and query IDE documents from sclang
                  • Support for the Atom text editor

                  Language-side news

                  • Improved Quark system and many new interesting Quarks.
                  • TempoClock: -beats can be set.
                  • An interface for key-value-pairs, see Key Value Pairs
                  • Refactored JITLib, see JITLib Changes in 3.7 (in particular dynamic channel expansion).
                  • QuartzComposerView

                  External Interfacing

                  There is an entirely new HID implementation: see Working with HID.

                  List of new methods and classes

                  • Collection: -collectCopy, Collection: -collectInPlace
                  • SimpleNumber: -lcm and SimpleNumber: -gcd have consistent interpretations of negative values and zero.
                  • Dictionary: -embedInStream can be customized from within the dictionary.
                  • Server: *remote Create a new Server instance corresponding to a server app running on a separate machine.

                  List of deprecated methods/classes, or that will be deprecated soon

                  • GeneralHID, GeneralHIDSpec, GeneralHIDDevice (replaced by the new HID system)

                  Server-side news

                  Apart from UDP, the TCP-protocol is now supported

                  List of new UGens

                  • NodeID (UGen that returns the current node id)
                  • Dconst

                  Improved or corrected behavior

                  • LinXFade2 (correct fading direction)
                  • Instances of PV_Copy are added automatically where necessary for parallel processing
                  • Env: *new accepts a new step2 shape, which steps to a value at the end of a shape
                  • UGen: -curvelin is now inverse of UGen: -lincurve

                  More operators work uniformly across sclang and scserver

                  The following operators have been added as ugens and work the same as in sclang:
                  unary operatorsrand, rand2, linrand, bilinrand, sum3rand, coin
                  binary operatorslcm, gcd, rrand, exprand

                  See Operators
