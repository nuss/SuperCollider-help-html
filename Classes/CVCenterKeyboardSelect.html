      CVCenterKeyboardSelect : Object
      Extension

      A GUI extension for CVCenterKeyboard that lets you choose among the SynthDefs stored with the keyboard. Can be controlled through OSC.
      Source: CVCenterKeyboardSelect.sc
      See also: CVCenterKeyboard, CVCenter

      Description

      A CVCenterKeyboardSelect may be used in connection with a CVCenterKeyboard and allows you to select the Synth that's to be controlled through your MIDI keyboard in a GUI or an external OSC controller (e.g. TouchOSC). A CVCenterKeyboardSelect can be created through the class' *new method described below described.

      Class Methods

      CVCenterKeyboardSelect.new(keyboardDef, tab: 'default', touchOSC, touchOSCPanel: 1)

      Create a new CVCenterKeyboardSelect instance. Note that any CVCenterKeyboardSelect will be dependant on an already existing CVCenterKeyboard instance.

      Arguments:

      keyboardDef

      a CVCenterKeyboard instance. The new CVCenterKeyboardSelect will automatically pick up the list of SynthDefs already added to the CVCenterKeyboard and add their names to the select's items.

      tab

      (optional) the method automatically creates a widget (a CVWidgetKnob) in CVCenter that is connected to the selection, letting you select a SynthDef through a slider. By default this widget will be placed in the 'default' tab of CVCenter.

      Returns:

      a new CVCenterKeyboardSelect

      CVCenterKeyboardSelect.allSelects

      Returns:

      an Event holding all currently existing CVCenterKeyboardSelect instances.

      Instance Methods

      .front

      Open the GUI

      .addOSC(listenerAddr, prevSynthCmd: "/prev_kb", nextSynthCmd: "/next_kb", nameCmd: "/select_kb_name", prefix: 1)

      Make keyboard SynthDefs selectable through OSC: A CVCenterKeyboardSelect makes use of CVWidgetKnobs (as part of CVCenter). A CVWidgetKnob not only has aGUI that lets you set values through sliders. It also provides easy OSC (and MIDI) connectabilty.

      When instantiating a CVCenterKeyboardSelect through the *new method you already create a CVWidgetKnob that lets you select a SynthDef for playing. This enough for the SC GUI on your computer but not yet suitable for an OSC controller that does not know about drop-down menus like e.g. TouchOSC.

      Hence, the method is designed for controllers that let you create your own interface and offer the following elements:

      push buttons
      buttons that upon click jump to 1 (or the "clicked" position) and on release back to 0 ("neutral" position)
      labels
      labels that display a text that can be set through OSC

      A select in TouchOSC could look e.g. like the following screenshot:


      A custom select in TouchOSC

      By calling addOSC on the CVCenterKeyboardSelect instance the following happens: Two more CVWidgetKnobs will be added to CVCenter. Both cause the select to step through its items, one descending, the other one ascending. Any time a switch to a new SynthDef happens the SynthDef's name is send to the controller, setting the label.

      Arguments:

      listenerAddr

      A NetAddr, reflecting the IP address and listening port of the device on which TouchOSC is installed.

      prevSynthCmd

      The OSC command name for the descending select button (labeled with a left arrow in the screenshot). E.g. '/previous_synth'.

      NOTE: If you have more than one CVCenterKeyboardSelect their commands must be unique. Otherwise you would switch all CVCenterKeyboardSelects at once.
      nextSynthCmd

      The OSC command name for the ascending select button (labeled with a right arrow in the screenshot)

      nameCmd

      The OSC command name under which the external controller will respond to the SynthDef names sent from SC.

      prefix

      TouchOSC interfaces can be organized in an arbitrary number of panels (called "pages"). The prefix reflects the panel or page on which the controlling elements are drawn. The prefix will be part of the OSC command names sent to and received from TouchOSC, e.g. "/1/next_kb". By default the prefix will be set to 1. If set to nil the prefix will be omitted in the OSC command names, e.g. "/next_kb".

      .removeOSC

      Removes all OSC listeners. Add new ones by calling -addOSC again.
