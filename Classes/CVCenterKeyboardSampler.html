      CVCenterKeyboardRecorder : Object
      Extension

      An extension to CVCenterKeyboard that lets you records sequences played on a MIDI keyboard.
      Source: CVCenterKeyboardSampler.sc
      See also: Pattern, Pdef, Pbind, Ppar, Ndef

      Description

      When creating a new CVCenterKeyboard you may have noticed the argument addSampler in CVCenterKeyboard: *new or CVCenterKeyboard: *newSynthDef which is by default set to true. I.e. if that argument is left unchanged a new CVCenterKeyboardRecorder instance, assigned to your CVCenterKeyboard instance will be created automatically. Now you should be able to "sample" any sequence you play on your keyboard by executing the -sample method once. Executing this method again will stop recording and the just recorded sequence should play instantly.

      Class Methods

      CVCenterKeyboardRecorder.new(keyboard)

      Create a new CVCenterKeyboardRecorder instance. It is recommended not to use this method directly as you will not be able to use shortcuts assigned to the CVCenterKeyboard instance like CVCenterKeyboard: -sample. Instead, if you have set the argument addSampler in CVCenterKeyboard: *new or CVCenterKeyboard: *newSynthDef to false, use CVCenterKeyboard: -addSampler.

      Arguments:

      keyboard

      The CVCenterKeyboard instance to which this CVCenterKeyboardRecorder instance shall be assigned.

      CVCenterKeyboardRecorder.all

      Returns:

      An Event, holding all CVCenterKeyboardRecorder instances under the names of the parent CVCenterKeyboard instances.

      Instance Methods

      .sample(onOff)

      Start or stop recording keyboard sequences. The CVCenterKeyboardRecorder instance should keep track of its state internally (is currently sampling or not), so, the argument onOff should be optional. However, it might be convenient to tell the recorder explicitly to start or stop in some situations.

      Arguments:

      onOff

      Optional - a Boolean, instructing the CVCenterKeyboardRecorder instance explicitly to start or stop recording.

      .clearSequences( ... keys)

      Clear recorded keyboard sequences, either all at once (if no keys are given) or only those denoted by the argument keys. Sequences are stored in a List that can be inspected by calling -pdef. However, as sequences are held in Pdefs as source of an Ndef they may also be addressed by the name/key of the regarding Ndef. If keys are given as Integers the method will internally query the correct Ndef at the given index. If the arument keys is omitted all sequences will be removed.

      Arguments:

      ... keys

      Optional - one or more Strings, Symbols or Integers addressing the sequence(s) to be removed from the CVCenterKeyboardRecorder instance.
